Техническое задание (ТЗ) на разработку проекта “SpringBoot_03 — Employee CRUD Service”
1. Общее описание проекта

Проект представляет собой backend-приложение, реализующее REST API для управления сотрудниками (Employee).
Приложение написано на Java 17 с использованием Spring Boot 3.3.1, Spring Web, Spring Data JPA, PostgreSQL,
Docker Compose, поддерживает логирование и тестирование с Testcontainers.

Система обеспечивает базовый CRUD-функционал: создание, чтение, обновление, удаление сотрудников.

3. Архитектура и структура проекта

Приложение следует многослойной архитектуре:

- Controller layer

Обрабатывает HTTP-запросы.

Валидирует входные данные (базовая проверка средствами Spring).

Возвращает ResponseEntity.

- Service layer

Содержит бизнес-логику CRUD.

Преобразует данные между DTO и Entity.

Работает с репозиторием.

- Repository layer

Работает с базой данных через Spring Data JPA.

Используется интерфейс JpaRepository.

- Database layer

Используется PostgreSQL.

Данные разворачиваются через Docker Compose.

Таблица employee создаётся автоматически (ddl-auto=create-drop).

4. Функциональные требования
4.1. CRUD для Employee
1. Получение сотрудника по ID

GET v1/api/employee/{id}
Ответ: 200 OK + JSON сотрудника
Ошибки: 404 NOT FOUND

2. Получение всех сотрудников

GET /api/employees
Ответ: 200 OK + JSON массив сотрудников

3. Создание сотрудника

POST /api/employees
Тело запроса:

{
  "id": null,
  "name": "Иван",
  "position": "Developer"
}


Ответ: 201 CREATED + созданный DTO

4. Обновление сотрудника

PUT /api/employees
Тело: EmployeeReqDto
Ответ: 200 OK

5. Удаление сотрудника

DELETE /api/employees/{id}
Ответ: 204 NO CONTENT

5. Описание данных
5.1. DTO: EmployeeDto
Integer id
String name
String position

5.2. Entity: Employee
id (PK, autogenerated)
name
position
creation_time (LocalDateTime, выставляется автоматически)

6. Требования к базе данных

База данных — PostgreSQL 13.
Схема создаётся автоматически Hibernate.

Docker Compose запускает сервисы:

postgres (порт 5432)

app (порт 8085)

В БД создаётся таблица:

employee (
    id SERIAL PRIMARY KEY,
    name VARCHAR,
    position VARCHAR,
    creation_time TIMESTAMP DEFAULT now()
)

10. Требования к качеству кода

Слои должны быть разделены (Controller → Service → Repository).

В сервисе должна быть валидация входных DTO.

DTO и Entity должны корректно преобразовываться.

Контроллер должен корректно обрабатывать исключения.

mapToDto() и mapToEntity() должны устанавливать все поля корректно
(в текущей версии есть ошибка — использование employeeDto.getId() вместо employee.getId()).
